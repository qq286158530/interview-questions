## 题目一：Redis 有哪些数据类型？分别在哪些场景下使用？

### 参考答案

Redis 主要有以下五种数据类型：

- **String（字符串）**：最简单类型，用于简单的 KV 缓存，如 `set name bingo`。
- **Hash（哈希）**：类似 map 结构，适合存储结构化对象（如用户信息），可操作单个字段，如 `hset person name bingo`。
- **List（列表）**：有序列表，可用于粉丝列表、评论列表，也可基于 `lrange` 实现高性能分页，或做简单消息队列（`lpush`/`rpop`）。
- **Set（无序集合）**：自动去重，适合全局去重场景，并支持交集（`sinter`）、并集（`sunion`）、差集（`sdiff`）操作。
- **Sorted Set（有序集合）**：带分数排序的去重集合，常用于排行榜场景，如 `zadd board 85 zhangsan`。

此外还有 Bitmaps、HyperLogLogs、Streams 等高级数据类型。

## 题目二：Redis 的持久化机制是什么？RDB 和 AOF 有什么区别？

### 参考答案

Redis 持久化有两种方式：

**RDB（Redis Database）**：周期性对数据执行快照，生成 `.rdb` 文件。优点是适合冷备恢复快、对主进程影响小；缺点是故障时会丢失最近一次快照以来的数据，且 fork 子进程时可能阻塞客户端。

**AOF（Append Only File）**：以追加方式将每条写命令记录到日志文件，通过回放命令重建数据。优点是数据安全性更高（可配置每秒 fsync）、支持日志重写恢复误删除；缺点是日志文件通常比 RDB 大，且写 QPS 相对较低。

**推荐策略**：同时开启两者，以 AOF 为主保证数据不丢失，用 RDB 做冷备快速恢复。

> 来源：[doocs/advanced-java - Redis 持久化机制](https://github.com/doocs/advanced-java/blob/main/docs/high-concurrency/redis-persistence.md)

## 题目三：Redis 集群是如何工作的？key 是如何寻址的？了解一致性 hash 算法吗？

### 参考答案

Redis cluster 通过 **hash slot** 机制实现数据分片：
- 整个集群有固定的 **16384 个 hash slot**
- 对每个 key 计算 `CRC16` 值，然后对 16384 取模，确定 key 所属的 slot
- 每个 master 节点持有部分 slot，增删节点时只需移动 hash slot，影响范围小

**分布式寻址算法演进**：
- **Hash 算法**：直接对节点数取模，节点宕机会导致大量缓存重建，不推荐。
- **一致性 Hash 算法**：将 key 和节点映射到环上，顺时针找到第一个节点。优点是节点增删影响范围小；缺点是容易出现数据分布不均（缓存热点），引入虚拟节点解决。
- **Redis cluster 的 hash slot 算法**：上述 hash slot 方案，客户端直接定位槽位，无需额外中间件。

Redis cluster 节点间通过 **gossip 协议**（ping/pong/meet/fail 消息）进行元数据交换，每个节点每秒 ping 10 次，选择 5 个最久未通信的节点。

> 来源：[doocs/advanced-java - Redis 集群模式原理](https://github.com/doocs/advanced-java/blob/main/docs/high-concurrency/redis-cluster.md)

## 题目四：什么是 Redis 的缓存雪崩、穿透和击穿？如何应对？

### 参考答案

**缓存雪崩**：大量缓存 key 在同一时刻全部失效，导致大量请求直接打到数据库。应对措施：事前做 Redis 高可用（主从+哨兵）；事中用本地缓存 + 限流降级；事后开启持久化快速恢复。

**缓存穿透**：恶意请求查询不存在的数据，缓存查不到、数据库也查不到，大量请求击穿数据库。解决方案：缓存空值（设置短过期时间）；使用布隆过滤器对请求做初筛。

**缓存击穿**：某个热点 key 过期瞬间，大量并发请求集中访问该 key，直接打到数据库。解决方案：热点数据永不过期；使用分布式互斥锁或本地锁限制只有少数请求重建缓存。

> 来源：[doocs/advanced-java - 缓存雪崩、穿透和击穿](https://github.com/doocs/advanced-java/blob/main/docs/high-concurrency/redis-caching-avalanche-and-caching-penetration.md)

## 题目五：Redis 和 Memcached 有什么区别？为什么 Redis 单线程却能支撑高并发？

### 参考答案

**主要区别**：
- Redis 支持更丰富的数据结构（String/Hash/List/Set/Sorted Set 等），Memcached 只支持 KV。
- Redis 原生支持集群模式，Memcached 需依赖客户端分片。
- 小数据存储 Redis 性能更高（单核）；大数据（100k+）Memcached 略有优势。

**Redis 单线程高并发的原因**：
- 纯内存操作，访问速度快
- 核心基于非阻塞 IO 多路复用（epoll/select/kqueue），无需多线程即可处理大量并发连接
- C 语言实现，执行效率高
- 单线程避免了上下文切换和锁竞争开销

注意：Redis 6.0 引入了多线程，但仅用于网络 IO 读写和协议解析，命令执行仍为单线程。

> 来源：[doocs/advanced-java - Redis 和 Memcached 的区别](https://github.com/doocs/advanced-java/blob/main/docs/high-concurrency/redis-single-thread-model.md)
