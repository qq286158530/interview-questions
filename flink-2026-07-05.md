## 题目一：Flink 的架构是怎样的？为什么说 Flink 是流处理的事实标准？

### 参考答案

Flink 架构核心分为 **三层**：

1. **Runtime 层**：分布式流处理引擎，采用 **Stream Operator** 模型，数据流通过 Operator DAG 进行处理。
2. **API 层**：提供 DataStream API（流处理）和 DataSet API（批处理，现已逐步废弃）。
3. **Library 层**：Table API / SQL、CEP 等高级封装。

**核心优势**：
- **真正的流优先（Streaming-first）**：批处理只是流处理的特例，DataStream API 可以处理有界和无界数据。
- **精确一次（Exactly-Once）语义**：通过 Checkpoint + 分布式快照（Chandy-Lamport 变体）实现端到端一致性。
- **低延迟高吞吐**：基于内存计算，支持增量检查点，状态后端（RocksDB）支持大状态。
- **时间语义丰富**：支持 Event Time、Processing Time、Ingestion Time，以及 Watermark 处理乱序事件。

> **来源**：[Apache Flink 官方文档](https://nightlies.apache.org/flink/flink-docs-release-1.18/docs/concepts/overview/)

## 题目二：Watermark（水位线）是什么？如何处理乱序和迟到数据？

### 参考答案

**Watermark（水位线）** 是 Flink 用于处理**乱序事件**的核心机制，本质是一个时间戳 T，表示"所有时间戳 ≤ T 的事件已经全部到达"。

**工作原理**：
- 当 Watermark T 到达某个窗口时，Flink 认为不会再有时间戳 < T 的事件，可以安全地触发窗口计算。
- 窗口的 `watermarkTime >= windowEndTime` 时触发计算。

**乱序处理策略**：

| 策略 | 配置方式 | 说明 |
|------|----------|------|
| 允许延迟 | `allowedLateness()` | 窗口触发后，保留状态一段时间（默认 0），期间迟到数据仍可触发增量计算 |
| 侧输出迟到数据 | `sideOutputLateData()` | 将迟到数据单独收集到侧输出流，避免丢失 |
| 直接丢弃 | 默认行为 | 丢弃时间戳 < watermark 的数据 |

**Watermark 生成方式**：
```java
// 有序流 Watermark 生成
WatermarkStrategy.forMonotonousTimestamps();

// 乱序流 Watermark 生成（最大延迟 5 秒）
WatermarkStrategy
    .<Event>forBoundedOutOfOrderness(Duration.ofSeconds(5))
    .withTimestampAssigner((event, ts) -> event.getTimestamp());
```

> **来源**：[Flink 窗口与 Watermark 官方文档](https://nightlies.apache.org/flink/flink-docs-release-1.18/docs/dev/datastream/event-time/generating_watermarks/)

## 题目三：Flink 的 Checkpoint 和 Savepoint 有什么区别？如何选择？

### 参考答案

| 维度 | Checkpoint | Savepoint |
|------|-----------|-----------|
| **触发方式** | Flink 自动周期性触发（由 JobManager 协调） | 手动触发（用户命令） |
| **用途** | 故障恢复，保证 exactly-once | 计划性备份、版本迁移 |
| **清理策略** | 作业删除时自动清理（可配置保留时间） | 需手动删除 |
| **格式** | 状态后端相关（二进制） | 路径化目录结构，兼容代码升级 |
| **依赖** | 需要开启 `env.enableCheckpointing()` | 需要配置 `SavepointDir` |

**实现原理**：
Flink 使用 **Chandy-Lamport 分布式快照算法变体**：通过在数据流中插入 Barrier（屏障），所有 Operator 收到所有输入的 Barrier 后做状态快照，然后异步上传到分布式存储（HDFS/S3）。

**最佳实践**：
```java
env.enableCheckpointing(60_000); // 每 60 秒一次
env.getCheckpointConfig().setMinPauseBetweenCheckpoints(30_000);
env.getCheckpointConfig().setCheckpointTimeout(120_000);
env.getCheckpointConfig().enableUnalignedCheckpoints();
```

> **来源**：[Flink Checkpoint 官方文档](https://nightlies.apache.org/flink/flink-docs-release-1.18/docs/ops/state/checkpoints/)

## 题目四：Flink 的状态后端（State Backend）有哪些？如何选型？

### 参考答案

Flink 有 **三种状态后端**：

### 1. HashMapStateBackend
- **存储方式**：状态存在 TaskManager JVM 堆内存中
- **适用场景**：状态较小、TTL 短、低延迟需求
- **优点**：访问速度最快（纯内存）
- **缺点**：受限于 JVM 堆大小，不支持增量 Checkpoint

### 2. EmbeddedRocksDBStateBackend
- **存储方式**：状态写入 RocksDB（嵌入式的 KV 数据库），RocksDB 数据存储在 TaskManager 本地磁盘
- **适用场景**：大状态、超大 keyed state（GB 级）、长 TTL
- **优点**：支持增量 Checkpoint，状态可超过内存大小
- **缺点**：序列化/反序列化开销，访问延迟高于纯内存

### 3. (已废弃) FsStateBackend
- 基于 HDFS/文件系统，仅做了解即可

**选型建议**：
- 状态 < 几 GB，考虑 **HashMapStateBackend**
- 状态 GB 级以上或需要 TTL 清理，考虑 **EmbeddedRocksDBStateBackend**
- 生产环境推荐 RocksDB + 增量 Checkpoint + 异步快照

```java
// 配置 RocksDB 状态后端
EmbeddedRocksDBStateBackend rocksDB = new EmbeddedRocksDBStateBackend();
env.setStateBackend(rocksDB);
```

> **来源**：[Flink State Backend 官方文档](https://nightlies.apache.org/flink/flink-docs-release-1.18/docs/ops/state/state_backends/)

## 题目五：Flink 窗口函数有哪些类型？滚动窗口、滑动窗口、会话窗口的区别？

### 参考答案

### 窗口类型

| 窗口类型 | 说明 | 典型场景 |
|----------|------|----------|
| **Tumbling Window（滚动窗口）** | 大小固定、不重叠 | 每分钟/每小时统计 |
| **Sliding Window（滑动窗口）** | 大小固定、可重叠 | 移动平均、趋势分析 |
| **Session Window（会话窗口）** | 动态窗口，间隙触发 | 用户行为分析、点击流 |
| **Global Window（全局窗口）** | 所有数据同一窗口，需自定义触发器 | 特殊业务逻辑 |

### 代码示例

```java
// 滚动窗口：每 5 分钟统计一次
window(TumblingEventTimeWindows.of(Time.minutes(5)))

// 滑动窗口：窗口大小 10 分钟，每 5 分钟滑动一次
window(SlidingEventTimeWindows.of(Time.minutes(10), Time.minutes(5)))

// 会话窗口：间隔超过 10 秒则开启新会话
window(EventTimeSessionWindows.withGap(Time.seconds(10)))
```

### 增量聚合 vs 全量聚合

- **增量聚合（ReduceFunction / AggregateFunction）**：每条数据到来时立即更新状态，输出单条结果，适合可递增计算（sum、avg、max）。
- **全量聚合（ProcessWindowFunction）**：窗口关闭时一次性处理所有数据，可访问窗口元信息，适合复杂计算（排序、中位数、TopN）。

> **来源**：[Flink 窗口函数官方文档](https://nightlies.apache.org/flink/flink-docs-release-1.18/docs/dev/datastream/operators/windows/)
